Elasticsearch Eğitimi

Şirket içi kapsamlı Elasticsearch eğitimi.

Neden Elasticsearch?

  • Geleneksel RDBMS'lerin full-text search ve büyük veri analizindeki zorlukları.
  • LIKE '%...%' sorgularının yetersizliği: Performans, alaka (relevance), dil desteği.
  • Yapısal olmayan (unstructured) ve yarı yapısal (semi-structured) verilerin (loglar, metinler, JSON dokümanları) artan önemi.
  • Gerçek dünya senaryoları: E-ticarette ürün arama, log analizi, anomali tespiti.

Elasticsearch Nedir?

Apache Lucene üzerine kurulu, dağıtık, RESTful arama ve analitik motorudur.

Temel Özellikleri:

  • Hız ve Ölçeklenebilirlik
  • Esneklik
  • Gerçek Zamanlıya Yakın (Near Real-Time) Sonuçlar

Yaygın Kullanım Alanları:

  • Full-Text Search (Gelişmiş site içi arama)
  • Log ve Metrik Analizi (Gözlemlenebilirlik - Observability)
  • İş Zekası (Business Intelligence) ve Analitik Raporlama
  • Güvenlik Analitiği (SIEM)

Temel Mimarî Kavramlar

Document (Doküman)

Verinin JSON formatındaki temel birimi.

Index (İndeks)

Benzer özelliklere sahip dokümanların koleksiyonu.

Node (Düğüm)

Cluster'ın bir parçası olan tek bir Elasticsearch sunucusu.

Cluster (Küme)

Bir veya daha fazla Node'dan oluşan yapı.

Shard (Parça)

Bir index'in yatay olarak bölündüğü parçalar.

Replica (Kopya)

Primary Shard'ların kopyaları. Yüksek erişilebilirlik sağlar.

Veri Yönetimi: Indexing ve Mapping

  • Doküman Yönetimi (CRUD Operasyonları): Index API, Get API, Update API, Delete API
  • Meta Alanlar: _index, _id, _version, _source, _score
  • Bulk API (_bulk): Toplu işlemler için performans ve verimlilik.
  • Mapping (Şema Yönetimi): Verinizin anayasası.
    • Dynamic Mapping: Otomatik tip algılama.
    • Explicit Mapping: Alan tiplerini manuel tanımlama.
    • Temel Veri Tipleri: text, keyword, sayısal tipler, date, boolean, ip, geo_point.
    • Mapping Parametreleri: index, analyzer, fields, dynamic.

Arama Sanatı: Query DSL ile Sorgulama

  • Arama Sorgusunun Anatomisi (_search API): URI Search vs Request Body Search.
  • Query Context vs Filter Context: Skorlama ve performans farkları.
  • Temel Sorgu Tipleri:
    • Full-Text Queries: match, match_phrase, multi_match.
    • Term-Level Queries: term, terms, range, exists.
  • Sorguları Birleştirme: bool Sorgusu (must, filter, should, must_not).

İleri Arama ve Aggregations

Arama Sonuçlarını Yönetme:

  • Sayfalama: from ve size.
  • Sıralama: sort.
  • Kaynak Filtreleme: _source.
  • Highlighting.

Aggregations (Veriyi Gruplama ve Analiz):

  • Metric Aggregations: min, max, avg, sum, stats, cardinality.
  • Bucket Aggregations: terms, range, date_range, histogram, date_histogram.
  • İç İçe Aggregation'lar (Sub-aggregations).

Elastic Stack Ekosistemi

Kibana

Veri görselleştirme ve keşif aracı. (Discover, Visualize, Dashboard).

Logstash

Sunucu taraflı veri işleme hattı. Veri toplama, dönüştürme ve gönderme.

Beats

Hafif, tek amaçlı veri göndericileri (Filebeat, Metricbeat vb.).

.NET Log Simülatörü

Eğitimde pratik uygulamalar için geliştirilmiş ASP.NET Core Razor Pages uygulaması.

Yetenekleri:

  • Index Template Yönetimi (application_logs).
  • products index'i ve örnek veri seti oluşturma.
  • Arka Planda Sürekli Log Yazma Servisi.
  • Geriye Dönük 30 Günlük Tarihsel Veri Oluşturma.
  • Elasticsearch bağlantı durumu ve küme sağlığı izleme.

// application_logs index template (kısaltılmış)
{
  "mappings": {
    "dynamic": "false",
    "properties": {
      "@timestamp": { "type": "date" },
      "service_name": { "type": "keyword" },
      "level": { "type": "keyword" },
      "message": { "type": "text" },
      // ... diğer alanlar
    }
  }
}

Hızlı Başlangıç

Projeyi yerel makinenizde çalıştırmak için adımlar:

  1. Depoyu klonlayın: git clone https://github.com/DogusTeknoloji/ElasticsearchTraining.git
  2. Docker ile Elasticsearch ve Kibana'yı başlatın: 
    
cd docker
docker-compose up -d
  3. Eğitim uygulamasını çalıştırın: 
    
cd src/ElasticTraining
dotnet run
  4. Kibana: http://localhost:5601
  5. Uygulama: https://localhost:5001
